Soft Computing

Fuzzy inference model

The inference block shows how continuous indicator values are transformed into memberships and rule strengths before producing a crisp action.

Problem analysis

Conventional screeners such as TradingView and Investing.com expose aggregated technical ratings, but they usually compress the reasoning into fixed votes. This dashboard keeps the summary while also exposing the fuzzy decision path, conflict between indicators, and the confidence of the final action.

Project design

The system follows a pipeline: market data acquisition, technical-indicator extraction, membership assignment, rule aggregation, defuzzified output, and an explanation layer. That makes the architecture modular enough for future AI summarization and alternative rule sets.

Functionality

The workspace supports multi-timeframe technicals, indicator tables, pivots, saved tickers, dynamic watchlists, signal-driver tracing, and a local summary block that can later be replaced by a model-generated narrative.

Relevance

Stock behavior is uncertain, nonlinear, and noisy, which fits soft computing well. Fuzzy logic handles partial truth directly: RSI can be partially oversold while momentum is only mildly bullish, so the output becomes a graded recommendation rather than a brittle yes-or-no trigger.
